2. CONFIGURACION
- git clone https://github.com/getsentry/sentry.git			- Descarga archivos
- git init								- Inicializa repo en la carpeta
- git remote add origin https://github.com/getsentry/sentry.git		- Conecta Rem.Rep.

3. DESCARGA AL LOCAL REPOSITORY
- git fetch origin		- Descarga ultimos cambios del remote al Working Directory
- git merge origin/master	- Absorve lo bajado a Origin/master en Master local

4. CARGUE DE CAMBIOS AL REMOTE REPOSITORY
- git fetch origin 			- Siempre antes de agregar commits descarga updates
- git push origin master

5. CONFIGURAR NUEVA TERMINAL PARA TRABAJO COLABORATIVO.
- Compartir folder contenedor del proyecto.
- En el nuevo computador conectar unidad de red a la ruta del directorio anterior.
- Agregar colaborardor (usuario de gitHub) desde la cuenta del repo original
- Intalar GIT en la nueva terminal.
- Generar llave para el repo al que vamos a controibuir.
- USAR COMANDO GIT LOG, GIT ADD -A, GIT COMMIT -M, GIT CHECKOUT, GIT FETCH, GIT MERGE ORIGIN/MASTER, GIT PUSH ORIGIN MASTER
